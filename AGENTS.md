# AGENTS.md

Guidance for agents working in this repository.

## Project Overview

SlipStack React is a small TypeScript React component library for horizontally sliding, stacking panes. The package entrypoint is `src/index.ts`; the main public component is `SlipStackContainer`.

## Tooling

- Use Node `>=22`.
- Use pnpm `>=11.8.0` through Corepack; this repository is configured as a pnpm workspace.
- Run pnpm commands as `corepack pnpm ...` so the pinned `packageManager` version is used even when a standalone `pnpm` shim is unavailable.
- Install dependencies with `corepack pnpm install`.
- Package JavaScript build output is generated by Vite into `dist/`; declaration output is generated by TypeScript and rewritten with `tsc-alias`.
- Tests run with Vitest in a jsdom environment.

## Common Commands

- `corepack pnpm test` - run the Vitest suite.
- `corepack pnpm test:watch` - run Vitest in watch mode.
- `corepack pnpm build` - build CommonJS, ESM, and declaration output.
- `corepack pnpm dev` - run Vite library build in watch mode.
- `corepack pnpm lint` - run ESLint over `src/**/*.{ts,tsx}` with fixes enabled.
- `corepack pnpm --filter scrollable-panes-example dev` - run the Vite example app from the workspace root.

## Repository Layout

- `src/` - library source and tests.
- `src/SlipStackContainer.tsx` - pane stack state, layout, scrolling, and imperative handle.
- `src/SlipStackPane.tsx` - pane data types and pane rendering component.
- `src/SlipStackTab.tsx` - vertical tab rendering.
- `src/useMeasure.ts` - measurement hook used by layout code.
- `example/` - Vite demo app consuming the package through the pnpm workspace.

## Coding Conventions

- Prefer the existing React functional component style with TypeScript types near the components that own them.
- Keep public exports centralized in `src/index.ts`.
- Preserve the package's path alias imports (`@/...`) where already used.
- The library currently uses inline style objects for component layout; keep related visual/layout changes close to the component unless a broader styling pattern is introduced intentionally.
- Use stable `id` values on pane data; pane navigation and replacement behavior depends on them.
- Be careful around scroll and measurement behavior. Layout calculations depend on viewport width, pane width, tab width, and `scrollLeft`.

## Testing Notes

- Add or update tests in `src/*.test.tsx` for behavior changes to pane opening, closing, tab visibility, scrolling, or imperative handle APIs.
- Vitest setup is in `vitest.setup.ts`; config is in `vitest.config.ts`.
- For user-visible component behavior, prefer Testing Library assertions over implementation details when practical.

## Dependency and Release Notes

- The project uses React 19 peer dependencies.
- This repository uses semantic-release and conventional commits.
- Install scripts are disabled in pnpm configuration; treat any dependency requiring install scripts as a supply-chain review item before changing that policy.
